Orden de uso de la API

  1. Crear Entrenador
  2. Crear Habilidad
  3. Crear Pokémon asignándolo a un entrenador y habilidad
  4. Consultar recursos (Entrenador, Habilidad, Pokémon)

Endpoints y ejemplos de payload

Entrenadores

  • GET /api/v1/entrenadores – Obtener todos los entrenadores
  • GET /api/v1/entrenadores/{id} – Obtener entrenador por ID
  • GET /api/v1/entrenadores/{id}/pokemons – Obtener todos los Pokémon de un entrenador
  • POST /api/v1/entrenadores – Crear nuevo entrenador
    Payload: 
    {
  "nombre": "Ash Ketchum"
}
  • PUT /api/v1/entrenadores/{id} – Actualizar entrenador
    Payload: 
    {
  "nombre": "Ash Ketchum"
}
  • DELETE /api/v1/entrenadores/{id} – Eliminar entrenador

Habilidades

  • GET /api/v1/habilidades – Obtener todas las habilidades
  • GET /api/v1/habilidades/{id} – Obtener habilidad por ID
  • GET /api/v1/habilidades/{id}/pokemons – Obtener todos los Pokémon asociados a una habilidad
  • POST /api/v1/habilidades – Crear nueva habilidad
    Payload: 
    {
  "nombre": "impactrueno"
}
  • PUT /api/v1/habilidades/{id} – Actualizar habilidad
    Payload: 
    {
  "nombre": "impactrueno mejorado"
}
  • DELETE /api/v1/habilidades/{id} – Eliminar habilidad

Pokémon

  • GET /api/v1/pokemons – Obtener todos los Pokémon
  • GET /api/v1/pokemons/{id} – Obtener Pokémon por ID
  • POST /api/v1/pokemons – Crear nuevo Pokémon
    Payload: 
    {
  "nombre": "Pikachu",
  "tipo": "eléctrico",
  "nivel": 5,
  "salud": 35,
  "entrenador": { "id": 1 },
  "habilidades": [
    { "id": 1 },
    { "id": 2 }
  ]
}
  • PUT /api/v1/pokemons/{id} – Reemplazar Pokémon
    Payload: 
    {
  "nombre": "Pikachu",
  "tipo": "eléctrico",
  "nivel": 6,
  "salud": 40,
  "entrenador": { "id": 2 },
  "habilidades": [
    { "id": 3 },
    { "id": 4 }
  ]
}
  • DELETE /api/v1/pokemons/{id} – Eliminar Pokémon
