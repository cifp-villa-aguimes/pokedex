1. Orden de uso de la API

  1. Crear Entrenador
  2. Crear Habilidad
  3. Crear Pokémon asignándolo a un entrenador y habilidad
  4. Consultar recursos (Entrenador, Habilidad, Pokémon)
  5. Simular Combate (API v2): iniciar combate, ejecutar turnos y consultar resultados

2. Entrenadores

  • GET /api/v1/entrenadores – Obtener todos los entrenadores
  • GET /api/v1/entrenadores/{id} – Obtener entrenador por ID
  • GET /api/v1/entrenadores/{id}/pokemons – Obtener Pokémon de un entrenador
  • POST /api/v1/entrenadores – Crear entrenador 
    {
  "nombre": "Ash Ketchum"
}
  • PUT /api/v1/entrenadores/{id} – Actualizar entrenador 
    {
  "nombre": "Ash Ketchum Updated"
}
  • PUT /api/v1/entrenadores/{id}/preserve – Actualiza solo el nombre, preservando Pokémon 
    {
  "nombre": "Ash Ketchum"
}
  • DELETE /api/v1/entrenadores/{id} – Eliminar entrenador

3. Habilidades

  • GET /api/v1/habilidades – Obtener todas
  • GET /api/v1/habilidades/{id} – Obtener por ID
  • GET /api/v1/habilidades/{id}/pokemons – Pokémon por habilidad
  • POST /api/v1/habilidades – Crear habilidad 
    {
  "nombre": "impactrueno"
}
  • PUT /api/v1/habilidades/{id} – Actualizar habilidad 
    {
  "nombre": "impactrueno mejorado"
}
  • DELETE /api/v1/habilidades/{id} – Eliminar habilidad

4. Pokémon

  • GET /api/v1/pokemons – Todos los Pokémon
  • GET /api/v1/pokemons/{id} – Por ID
  • POST /api/v1/pokemons – Crear Pokémon 
    {
  "nombre":"Pikachu",
  "tipo":"eléctrico",
  "nivel":5,
  "salud":35,
  "entrenador":{"id":1},
  "habilidades":[{"id":1},{"id":2}]
}
  • PUT /api/v1/pokemons/{id} – Reemplazar Pokémon 
    {
  "nombre":"Pikachu",
  "tipo":"eléctrico",
  "nivel":6,
  "salud":40,
  "entrenador":{"id":2},
  "habilidades":[{"id":3},{"id":4}]
}
  • DELETE /api/v1/pokemons/{id} – Eliminar Pokémon
  • Búsqueda avanzada (v2)
  • GET /api/v2/pokemons/buscar?tipo=agua
  • GET /api/v2/pokemons/buscar?nivelMin=10&nivelMax=50
  • GET /api/v2/pokemons/buscar?sortBy=salud&order=desc
  • GET /api/v2/pokemons/buscar?tipo=fuego&nivelMin=5&nivelMax=30&sortBy=nivel&order=asc

5. Combates (API v2)

  • POST /api/v2/combates?playerAId={idA}&playerBId={idB} – Inicia combate bidireccional
  • GET /api/v2/combates/en-curso – Lista combates en curso
  • POST /api/v2/combates/{combateId}/turno – Ejecuta siguiente turno
  • GET /api/v2/combates/{combateId} – Estado actual (saludA, saludB, turnoActual, estado)
  • GET /api/v2/combates/{combateId}/turnos – Histórico de turnos
  • GET /api/v2/combates/turnos/por-atacante?atacanteId={pokemonId} – Filtra turnos por atacante
