Orden de uso de la API

  1. Crear Entrenador
  2. Crear Habilidad
  3. Crear Pokémon asignándolo a un entrenador y habilidad
  4. Consultar recursos (Entrenador, Habilidad, Pokémon)

Endpoints y ejemplos de payload

Entrenadores

  • GET /api/v1/entrenadores – Obtener todos los entrenadores
  • GET /api/v1/entrenadores/{id} – Obtener entrenador por ID
  • GET /api/v1/entrenadores/{id}/pokemons – Obtener todos los Pokémon de un entrenador
  • POST /api/v1/entrenadores – Crear nuevo entrenador
    Payload: 
    {
  "nombre": "Ash Ketchum"
}
  • PUT /api/v1/entrenadores/{id} – Actualizar entrenador
    Payload: 
    {
  "nombre": "Ash Ketchum"
}
  • DELETE /api/v1/entrenadores/{id} – Eliminar entrenador

Habilidades

  • GET /api/v1/habilidades – Obtener todas las habilidades
  • GET /api/v1/habilidades/{id} – Obtener habilidad por ID
  • GET /api/v1/habilidades/{id}/pokemons – Obtener todos los Pokémon asociados a una habilidad
  • POST /api/v1/habilidades – Crear nueva habilidad
    Payload: 
    {
  "nombre": "impactrueno"
}
  • PUT /api/v1/habilidades/{id} – Actualizar habilidad
    Payload: 
    {
  "nombre": "impactrueno mejorado"
}
  • DELETE /api/v1/habilidades/{id} – Eliminar habilidad

Pokémon

  • GET /api/v1/pokemons – Obtener todos los Pokémon
  • GET /api/v1/pokemons/{id} – Obtener Pokémon por ID
  • POST /api/v1/pokemons – Crear nuevo Pokémon
    Payload: 
    {
  "nombre": "Pikachu",
  "tipo": "eléctrico",
  "nivel": 5,
  "salud": 35,
  "entrenador": { "id": 1 },
  "habilidades": [
    { "id": 1 },
    { "id": 2 }
  ]
}
  • PUT /api/v1/pokemons/{id} – Reemplazar Pokémon
    Payload: 
    {
  "nombre": "Pikachu",
  "tipo": "eléctrico",
  "nivel": 6,
  "salud": 40,
  "entrenador": { "id": 2 },
  "habilidades": [
    { "id": 3 },
    { "id": 4 }
  ]
}
  • DELETE /api/v1/pokemons/{id} – Eliminar Pokémon
  • Búsqueda avanzada de v2
  • GET /api/v2/pokemons/buscar – Buscar todos los Pokémon con filtros por defecto
  • GET /api/v2/pokemons/buscar?tipo=agua – Filtrar por tipo “agua”
  • GET /api/v2/pokemons/buscar?nivelMin=10&nivelMax=50 – Filtrar por nivel entre 10 y 50
  • GET /api/v2/pokemons/buscar?sortBy=salud&order=desc – Ordenar por salud descendente
  • GET /api/v2/pokemons/buscar?tipo=agua&nivelMin=5&nivelMax=30&sortBy=nivel&order=asc – Combinación de filtros y ordenación

Combates (API v2)

  • POST /api/v2/combates?playerAId={idA}&playerBId={idB} – Inicia un nuevo combate (bidireccional) y devuelve el ID.
  • GET /api/v2/combates/en-curso – Lista todos los combates que están en curso.
  • POST /api/v2/combates/{combateId}/turno – Ejecuta el siguiente turno alternando atacante y devuelve el turno ejecutado.
  • GET /api/v2/combates/{combateId} – Obtiene el estado actual del combate (saludA, saludB, turnoActual, estado).
  • GET /api/v2/combates/{combateId}/turnos – Lista todos los turnos ejecutados en el combate.
  • GET /api/v2/combates/turnos/por-atacante?atacanteId={pokemonId} – Filtra y devuelve los turnos donde el Pokémon ataca.
